# Drishti System Architecture

## 1. High-Level Workflow

The overall workflow of Drishti is as follows:

1. The police officer enters a query through the chatbot.
2. The AI understands the user's intent.
3. The system determines what information is required.
4. Relevant crime records are retrieved from the database.
5. The analytics engine processes the retrieved data.
6. The AI generates an evidence-backed response.
7. The response is presented to the user along with charts, maps, or relationship graphs when applicable.

## 2. System Components

Drishti consists of the following major components:

### 1. Chat Interface
Provides a conversational interface where investigators can ask questions using text or voice.

### 2. Backend API
Receives requests from the frontend, processes them, and coordinates communication between the AI, database, and analytics engine.

### 3. AI Engine
Understands user queries, extracts the required information, and generates natural language responses based on retrieved data.

### 4. Crime Database
Stores structured crime records including FIRs, accused, victims, locations, investigation status, and crime history.

### 5. Crime Analytics Engine
Analyzes crime data to identify trends, hotspots, relationships, and other investigative insights.

### 6. Visualization Module
Displays charts, maps, timelines, and criminal relationship graphs to help investigators understand the results.

### 7. Catalyst Cloud Services
Provides hosting, authentication, database services, AI services, voice processing, and deployment infrastructure.
